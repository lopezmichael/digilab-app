**Goal:** Migrate from DuckDB/MotherDuck to Neon PostgreSQL as the single database for all environments.

**Architecture:** Single Neon Postgres database replaces both local DuckDB and cloud MotherDuck. R Shiny app uses `pool` + `RPostgres` instead of `duckdb`. Python sync scripts use `psycopg2` instead of `duckdb`. All environments share one database.

## Task 4: R Connection Module

**Files:**
- Rewrite: `R/db_connection.R`

**Context:** This is the core change. Replace DuckDB connection logic with a `pool`-based Postgres connection. The pool object is created once at app startup and shared across the entire app.

**Step 1:** Rewrite `R/db_connection.R`:

```r
# =============================================================================
# Database Connection Module
# DigiLab - https://digilab.cards/
#
# Uses Neon PostgreSQL via pool + RPostgres.
# Single connection pool shared across the app.
# =============================================================================

library(DBI)
library(pool)
library(RPostgres)

# Create the connection pool (called once at app startup)
create_db_pool <- function() {
  host <- Sys.getenv("NEON_HOST")
  dbname <- Sys.getenv("NEON_DATABASE", "neondb")
  user <- Sys.getenv("NEON_USER")
  password <- Sys.getenv("NEON_PASSWORD")

  if (host == "" || password == "") {
    stop("NEON_HOST and NEON_PASSWORD environment variables are required. See .env.example")
  }

  tryCatch({
    p <- dbPool(
      RPostgres::Postgres(),
      host = host,
      dbname = dbname,
      user = user,
      password = password,
      port = 5432,
      sslmode = "require",
      minSize = 1,
      maxSize = 5
    )
    message("Connected to Neon PostgreSQL: ", dbname, " @ ", host)
    p
  }, error = function(e) {
    stop("Database connection failed: ", e$message)
  })
}

# Close the pool (called on app shutdown)
close_db_pool <- function(p) {
  if (!is.null(p)) {
    tryCatch({
      poolClose(p)
      message("Database pool closed")
    }, error = function(e) {
      message("Error closing pool: ", e$message)
    })
  }
}

# Schema initialization (for fresh databases)
init_schema <- function(pool, schema_path = "db/schema_postgres.sql") {
  if (!file.exists(schema_path)) {
    stop("Schema file not found: ", schema_path)
  }

  schema_sql <- readLines(schema_path, warn = FALSE)
  schema_sql <- schema_sql[!grepl("^\\s*--", schema_sql)]
  schema_sql <- paste(schema_sql, collapse = "\n")
  statements <- strsplit(schema_sql, ";")[[1]]

  success_count <- 0
  for (stmt in statements) {
    stmt <- trimws(stmt)
    if (stmt == "" || nchar(stmt) < 5) next
    tryCatch({
      dbExecute(pool, paste0(stmt, ";"))
      success_count <- success_count + 1
    }, error = function(e) {
      if (!grepl("already exists", e$message, ignore.case = TRUE)) {
        warning("FAILED: ", substr(gsub("\\s+", " ", stmt), 1, 60), " | ", e$message)
      }
    })
  }
  message("Schema initialized (", success_count, " statements)")
}

# Verify all required tables exist
check_schema <- function(pool) {
  required <- c("stores", "players", "deck_archetypes",
                "archetype_cards", "tournaments", "results")
  existing <- dbListTables(pool)
  missing <- setdiff(required, existing)
  if (length(missing) > 0) {
    warning("Missing tables: ", paste(missing, collapse = ", "))
    return(FALSE)
  }
  message("All tables present")
  TRUE
}

# Simple query helper
query <- function(pool, sql) {
  dbGetQuery(pool, sql)
}

# Log a data operation
log_operation <- function(pool, source, action, status, records = 0, error = NULL) {
  dbExecute(pool,
    "INSERT INTO ingestion_log (source, action, status, records_affected, error_message)
     VALUES ($1, $2, $3, $4, $5)",
    params = list(source, action, status, records, error))
}
```

**Key changes from original:**
- `connect_db()` / `connect_motherduck()` / `connect_local()` → `create_db_pool()`
- `disconnect()` → `close_db_pool()`
- No more DuckDB-specific code (INSTALL, LOAD, ATTACH, USE)
- `log_operation` uses `$1, $2` Postgres placeholders instead of `?` (RPostgres uses `$N` syntax)

**Important:** RPostgres uses `$1, $2, $3...` for parameterized queries, NOT `?`. This affects ALL parameterized queries in the entire codebase. DBI's `?` placeholders work with the `duckdb` driver but NOT with `RPostgres`.

**Step 2:** Commit.

```bash

git commit -m "feat: replace DuckDB connection with Neon pool"
```

## Task 8: Convert ? Placeholders to $N (RPostgres Syntax)

**Files:** All files with parameterized queries (every server/*.R, R/db_connection.R)

**Context:** DuckDB's R driver uses `?` for parameter placeholders. RPostgres uses `$1, $2, $3...` numbered placeholders. Every parameterized query must be converted.

**Step 1:** In every file, find all `params = list(...)` calls and convert the corresponding SQL from `?` to `$1, $2, $3...` etc.

Example conversions:

```r
# Before (DuckDB):
dbGetQuery(con, "SELECT * FROM players WHERE player_id = ?", params = list(42))

# After (Postgres):
dbGetQuery(db_pool, "SELECT * FROM players WHERE player_id = $1", params = list(42))
```

```r
# Before:
dbExecute(con, "UPDATE stores SET name = ?, city = ? WHERE store_id = ?",
  params = list(name, city, store_id))

# After:
dbExecute(db_pool, "UPDATE stores SET name = $1, city = $2 WHERE store_id = $3",
  params = list(name, city, store_id))
```

**Important:** `build_filters_param()` in `server/shared-server.R` dynamically builds SQL with `?` placeholders. This must be converted to use a counter for `$N` numbering.

Update `build_filters_param()` to track parameter index:
```r
# Instead of appending "AND t.format = ?" append "AND t.format = $N"
# where N increments for each parameter added
```

**Step 2:** Also convert `LOWER(x) LIKE LOWER(?)` to `x ILIKE $N` wherever it appears (build_filters_param).

**Step 3:** Commit.

```bash
git add server/ R/
git commit -m "refactor: convert ? placeholders to \$N for RPostgres"
```
